#TitleTypeSeverity StatusScoreDescriptionMitigations
No threats

App esterna (Actor)

Description: Applicazione client esterna che comunica verso l'API pubblica

#TitleTypeSeverity StatusScoreDescriptionMitigations
1 Spoofing dell'App esterna Spoofing High Open Un attaccante potrebbe impersonare l'App esterna inviando richieste API non autorizzate sfruttando l'assenza di mutua autenticazione. Implementare autenticazione client tramite certificati mTLS o API key con rotazione periodica.
2 Repudiation delle richieste API Repudiation Medium Open L'App esterna potrebbe negare di aver effettuato richieste in assenza di un meccanismo di audit trail affidabile. Abilitare logging strutturato con timestamp firmati su ogni richiesta in ingresso. Conservare i log su storage immutabile.

#TitleTypeSeverity StatusScoreDescriptionMitigations
No threats

App Service (Process)

Description: Servizio applicativo interno nel perimetro cliente che espone l'API

#TitleTypeSeverity StatusScoreDescriptionMitigations
5 Elevation of Privilege tramite bypass autorizzazione Elevation of privilege Critical Open Un attaccante autenticato potrebbe sfruttare logiche di autorizzazione insufficienti (IDOR, privilege escalation) per accedere a risorse di altri utenti o funzioni privilegiate. Implementare RBAC con verifica dell'ownership a livello di ogni endpoint. Eseguire revisione del codice focalizzata su authorization checks.
6 Denial of Service sull'endpoint API Denial of service High Open L'endpoint API esposto pubblicamente può essere bersaglio di attacchi volumetrici o di tipo slowloris che ne compromettono la disponibilità. Configurare rate limiting, throttling e circuit breaker. Integrare Azure DDoS Protection o equivalente WAF davanti all'App Service.
7 Information Disclosure tramite error messages Information disclosure Medium Open Messaggi di errore dettagliati (stack trace, connection string, versioni componenti) possono esporre informazioni sensibili sull'infrastruttura interna. Configurare gestione centralizzata degli errori con messaggi generici verso il client. Loggare i dettagli solo internamente.

HTTP Request (Data Flow)

Description: Richiesta HTTPS dall'App esterna verso Internet

#TitleTypeSeverity StatusScoreDescriptionMitigations
3 Tampering dei dati in transito Tampering High Open Un attaccante man-in-the-middle potrebbe intercettare e modificare il payload delle richieste HTTP se il canale non è adeguatamente protetto. Forzare TLS 1.2+ su tutti i canali. Implementare certificate pinning lato client. Verificare l'integrità del payload con HMAC.

API (Data Flow)

Description: Chiamata API REST verso l'App Service attraverso il perimetro cliente

#TitleTypeSeverity StatusScoreDescriptionMitigations
4 Information Disclosure su canale API non cifrato Information disclosure High Open Se il canale API non utilizza TLS, dati sensibili (token, payload, PII) possono essere intercettati da attori sulla rete pubblica. Imporre HTTPS obbligatorio con redirect automatico da HTTP. Configurare HSTS con preload. Utilizzare TLS 1.3 dove possibile.
8 Injection tramite input non validato Tampering High Open Payload API contenenti input malevoli (SQL injection, command injection, XXE) non validati lato server possono compromettere il sistema backend. Implementare validazione e sanitizzazione degli input server-side. Utilizzare query parametrizzate. Adottare un WAF davanti all'endpoint.
